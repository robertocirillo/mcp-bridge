# Security Policy

## Reporting A Vulnerability

Please do not open public issues for suspected security problems.

Use GitHub's private vulnerability reporting for this repository when available. If that is not available, contact the maintainer privately on GitHub (`@robertocirillo`) and include:

- a short description of the issue
- affected versions or configuration
- reproduction steps or a proof of concept
- any suggested remediation or mitigation

## Scope

Security reports are most useful when they cover issues such as:

- authentication, authorization, or tenant-isolation flaws
- session or guardrail bypasses
- sensitive data exposure
- unsafe direct tool execution paths
- dependency or supply-chain risk with a concrete impact

Please share enough detail to reproduce the problem. Coordinated private disclosure is preferred until a fix is available.
